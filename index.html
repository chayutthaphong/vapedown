Program Length (quit within)

Evidence-based guidance

Research favours shorter tapers (2–4 weeks). Behavioural trials set quit dates 0–5 weeks out (Lindson review); one common schedule cuts baseline cigarettes/day by 25% over 4 weeks (Skelton). Longer programmes see more drop-out (RedPharm compared 4 vs 16 weeks; participants stayed ~6 weeks on average). What predicts success most is sticking to the schedule, not length. → If you can tolerate it, 4 weeks is the recommended choice; pick 6–8 only if withdrawal feels too harsh.

Taper Plan (your ceiling)

Ceiling = baseline puffs/day (from your FTND + PS-ECDI scores) × weekly reduction. The ceiling is a hard limit, not a target — staying well below it is the goal. It is set deliberately far under typical real-world use (POD users average ~123 puffs/day, EMIT study) and steps down to zero. Your current week is highlighted.

Disclaimer & References

Disclaimer: This application provides a structural framework for nicotine tapering and is intended for personal data collection and longitudinal tracking. It does not constitute formal medical advice, diagnosis, or behavioral therapy. Please consult healthcare professionals for personalized guidance.

Clinical rationale: Pre-filled closed-system pod devices cannot be diluted, so dependence reduction relies on behavioural frequency modification, interval extension, and trigger decoupling (CBT principles). Dependence is assessed with the FTND and the Penn State ECDI; the weekly puff ceiling is a deliberately quit-oriented target set below typical real-world consumption and stepped down to zero over the chosen program length.

Ceiling vs daily challenge: Two targets are shown together. The weekly ceiling is a fixed hard limit for the current program week and steps down each week to zero. The daily challenge is a dynamic, tighter target defined as challenge = min( weekly ceiling, yesterday's actual puffs ) × 0.9 — it is always 10% below your reference point. In practice: on the first day (no prior day) the reference is the ceiling, so the challenge is the ceiling × 0.9; if yesterday you stayed below the ceiling, the reference is what you actually used, so the challenge keeps ratcheting down as you improve; if yesterday you reached or exceeded the ceiling, the reference is capped at the ceiling, so the challenge becomes the ceiling × 0.9. When the program moves to a new week and the ceiling drops, the same formula uses the new, lower ceiling automatically. The ceiling guarantees a minimum rate of reduction; the challenge always pushes 10% further. Both are pragmatic design choices, not validated dosing protocols.
